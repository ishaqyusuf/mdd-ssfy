## Purpose
This is the Phase 0 ownership matrix for the inventory correctness cutover. It maps each non-negotiable invariant to code owners, current evidence, status, and the remaining proof gate. The goal is to prevent later phases from continuing with any unowned correctness rule.

## Status Key
- `proven-package`: focused package/API tests or deterministic code evidence exist, but browser/operator evidence may still be needed.
- `implemented-needs-proof`: code path exists, but current proof is incomplete for full cutover confidence.
- `partial`: meaningful coverage exists, but one or more important mutation paths still need hardening or stronger evidence.
- `pending`: no sufficient current proof yet.
