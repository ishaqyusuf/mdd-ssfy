## Worker Usability Review

Sales reps now have a clearer dealer request queue and the dealer approval path has real QA evidence. The weak point is still approval ergonomics: a `window.prompt` is not a reliable office workflow for reviewed delivery/ship costs, especially when staff need to justify the charge later.

Warehouse and dispatch users still face split proof semantics. Mobile completion requires signature/photo upload work before the completion task, but the API boundary accepts dispatch mutation and upload routes publicly. A mixed-skill team needs one clear result: either the delivery is completed with proof attached, or it is still open with a recoverable pending proof state.

Inventory workers have many improved protected inventory routes and table surfaces, but Brain still says the cutover is not release-clean. The product should keep the current guardrail copy prominent until the stopped repair loop and browser proof matrix are explicitly resumed or scoped.

## Customer And Dealer Experience Review

The dealership program has improved substantially: Brain now records customer/internal PDFs as separate surfaces, customer-paid dealer orders no longer clear the internal GND payable, and dealer dashboard payment behavior was QA'd on `00003DPP`.

Customer trust risk remains in document delivery. The live Share action is not recipient-aware and uses a fixed phone number. Until the planned short-link WhatsApp/SMS flow lands, staff can accidentally send a valid document token to the wrong destination.

Dealer experience risk remains around old paths. The dealer UI now uses `Request order`, but the direct conversion tRPC endpoint still exists. If any stale client, script, or overlooked control calls it, the workflow can bypass sales review.

## Door-Manufacturing Workflow Gaps

- Delivery/ship cost review should be a structured approval step with cost, note, and required reason, not a browser prompt.
- Quote-to-order approval should enforce one visible state model: draft quote, requested quote, approved order, rejected quote, or revision needed.
- Dispatch proof should attach to the same completion transaction or show a recoverable "proof uploaded, completion failed" state.
- Inventory readiness should stay explicit before production/dispatch cutover: missing sales, mapping-blocked rows, and shipment/allocation drift are still unresolved by Brain.
- Customer/dealer document delivery should use the actual customer/dealer phone/email, short links, and per-channel send evidence.
